Zookeeper+Hadoop+Hbase整合平台搭建（伪分布模式）

JDK的卸载与安装

Zookeeper的安装

Hadoop的安装

Hbase的安装

伪分布式平台架构

首先我们需要下载搭建平台所需要的资源包并放入指定文件目录下，这里博主将包放入了根目录下的software文件中。

资源获取地址：https://pan.baidu.com/s/1EW8t5szEkG-1Uk6iqeRQbQ 提取码：2a3c

通过xftp 6 将资源包传输到 /software 目录下

我们可以在CentOS系统里查看传输好的文件

一.JDK的卸载与安装

从普通用户进入root 用户，在root用户卸载java（为了方便起见，博主将用户名设为hadoop,请不要与下面的hadoop用户混淆)

[hadoop@promote~]$su - root

  • 卸载jdk
[root@promote ~]#rpm -qa|grep jdk      查看jdk包
[root@promote ~]#rpm -qa|grep java     查看java包
[root@promote ~]#rpm -e java-XXXX      用rpm命令删除包
[root@promote ~]#yum remove java-XXXX  用yum命令删包，有些包会报“依赖项错误”，这时用yum就可以解决问题

首先查看系统中的jdk包,发现有4个压缩包，我们要做的就是将它们删除

[root@promote ~]#rpm -qa|grep jdk

删除jdk包，rpm -e + 包名，最好是一个一个删除，不要用 rpm -e * 或 yum remove * 一次性将包删除，一次性删除可能会导致后面坏境的搭建出错

用 rpm -e + 包名，发现报错，此时用yum remove + 包名（推荐）

[root@promote ~]#yum remove java-XXXX

​ 删除完后，用 rpm -qa|grep jdk 会发现输出为空，说明jdk压缩包已删除

​ 重复上述步骤删除java包

  • 安装jdk

卸载完jdk和java后，我们在root用户下新建java目录,并进入java目录

[root@promote ~]#mkdir /java
[root@promote ~]#cd /java

我们可以查看传到/software目录下压缩包

接下来就是解压software目录下的 jdk-8u201-linux-x64.tar.gz

[root@promote java]#tar -zxf /software/jdk-8u201-linux-x64.tar.gz

  • 修改java坏境变量

进入hadoop用户，如果诸位系统中没有hadoop用户，可以新建一个hadoop用户,用 useradd hadoop,这样的好处就是当我们需要进行hadoop相关操作时就可以进入该用户，不需要再找了，同时也避免将其框架弄坏。

[root@promote java]#su - hadoop

打开 .bash_profile 配置文件，添加java环境变量

vi .bash_profile
#Java environment
JAVA_HOME=/java/jdk1.8.0_201
PATH=$JAVA_HOME/bin:$PATH
export JAVA_HOME CLASSPATH PATH

source 文件

[hadoop@promote ~]$source .bash_profile

输入java -version,如果出现图片中的信息，说明配置成功。否则删除root下的java文件，重新进行配置

二.Zookeeper的安装

  • zookeeper软件包解压

从hadoop用户退出，进入root用户

[root@promote ~]#su - root

在根目录下创建zookeeper目录

[root@promote ~]#mkdir /zookeeper
[root@promote ~]#cd /

将zookeeper的权限变更给hadoop

[root@promote /]#chown hadoop /zookeeper

进入hadoop用户并切换到zookeeper目录

[root@promote /]#su - hadoop
[hadoop@promote ~]$cd /zookeeper

在当前目录下解压缩zookeeper包

[hadoop@promote zookeeper]$tar -zxf /software/zookeeper-3.4.13.tar.gz
  • zookeeper坏境变量设定

配置zookeeper的环境变量，将下面的变量添加进去

“~”表示用户主目录

[hadoop@promote zookeeper]$vi ~/.bash_profile     进入.bash_profile
#zookeeper environment
ZOOKEEPER_HOME=/zookeeper/zookeeper-3.4.13
PATH=$ZOOKEEPER_HOME/bin:$PATH
export ZOOKEEPER_HOME PATH

source 文件

[hadoop@promote zookeeper]$source ~/.bash_profile

验证坏境变量是否配置成功

[hadoop@promote zookeeper]$echo $PATH
  • zookeeper伪分布模式参数配置

创建3个data和logs目录

[hadoop@promote zookeeper]$mkdir -p zookeeper1/data
[hadoop@promote zookeeper]$mkdir -p zookeeper2/data
[hadoop@promote zookeeper]$mkdir -p zookeeper3/data
[hadoop@promote zookeeper]$mkdir -p zookeeper1/logs
[hadoop@promote zookeeper]$mkdir -p zookeeper2/logs 
[hadoop@promote zookeeper]$mkdir -p zookeeper3/logs

在3个data目录里创建myid文件

[hadoop@promote zookeeper]$echo '1' > zookeeper1/data/myid
[hadoop@promote zookeeper]$echo '2' > zookeeper2/data/myid
[hadoop@promote zookeeper]$echo '3' > zookeeper3/data/myid

查看myid里的id是否一一对应

[hadoop@promote zookeeper]$cat zookeeper1/data/myid
[hadoop@promote zookeeper]$cat zookeeper2/data/myid
[hadoop@promote zookeeper]$cat zookeeper3/data/myid

进入zookeeper-3.4.13conf目录

[hadoop@promote zookeeper]$cd /zookeeper/zookeeper-3.4.13/conf

用zoo_sample.cfg去生成zoo1.cfg的congfig文件，然后对zoo1.cfg文件进行修改与添加

[hadoop@promote conf]$cp zoo_sample.cfg zoo1.cfg   复制
[hadoop@promote conf]$vi zoo1.cfg    进入文件

将文件中的dataDir和clientPort两个参数注释掉，因为它们会影响后面的配置，同时将变量添加进去

dataDir=/zookeeper/zookeeper1/data      #指定Zookeeper的data路径
dataLogDir=/zookeeper/zookeeper1/logs   #指定Zookeeper的log路径
clientPort=2181   #指定Client连接端口，对外唯一
server.1=localhost:2888:3888     #指定server node，及其服务端口。localhost可切换为主机名
server.2=localhost:2889:3889
server.3=localhost:2890:3890

复制zoo1.cfg生成zoo2.cfg zoo3.cfg

[hadoop@promote conf]$cp zoo1.cfg zoo2.cfg
[hadoop@promote conf]$cp zoo1.cfg zoo3.cfg
[hadoop@promote conf]$vi zoo2.cfg
[hadoop@promote conf]$vi zoo3.cfg

注意，要修改zoo2.cfg和zoo3.cfg的dataDir、dataLogDir和clientPort中的序号

启动服务

[hadoop@promote conf]$zkServer.sh start zoo1.cfg
[hadoop@promote conf]$zkServer.sh start zoo2.cfg
[hadoop@promote conf]$zkServer.sh start zoo3.cfg

查看工作进程

[hadoop@promote conf]$jps

查看工作状态

[hadoop@promote conf]$zkServer.sh status zoo1.cfg
[hadoop@promote conf]$zkServer.sh status zoo2.cfg
[hadoop@promote conf]$zkServer.sh status zoo3.cfg

出现上述的查询结果说明zookeeper已经配置完成

三.Hadoop的安装

  • hadoop 软件包解压

切换到root的主目录

[hadoop@promote conf]$su - root

在根目录下创建hadoop目录，并将hadoop目录的权限给hadoop用户，然后切换到hadoop用户，进入hadoop目录，同时在当前目录下解压hadoop包

[root@promote ~]#mkdir /hadoop
[root@promote ~]#cd /
[root@promote /]#chown hadoop /hadoop
[root@promote /]#su - hadoop
[hadoop@promote ~]$cd /hadoop
[hadoop@promote hadoop]$tar -zxf /software/hadoop-2.7.7.tar.gz
  • hadoop坏境变量设定

添加hadoop环境变量

[hadoop@promote hadoop]$vi ~/.bash_profile
#hadoop environment
HADOOP_HOME=/hadoop/hadoop-2.7.7
PATH=$HADOOP_HOME/bin:$HADOOP_HOME/sbin:$PATH
export HADOOP_HOME PATH
[hadoop@promote hadoop]$source ~/.bash_profile

验证配置是否成功

[hadoop@promote hadoop]$echo $PATH

  • hadoop参数配置

创建 tmp data name 三个文件目录

[hadoop@promote hadoop]$mkdir tmp data name

配置本机免密码登录

[hadoop@promote hadoop]$cd ~
[hadoop@promote ~]$ssh-keygen -t rsa   输入三个enter

[hadoop@promote ~]$cd  .ssh 
[hadoop@promote .ssh]$cp id_rsa.pub authorized_keys
[hadoop@promote .ssh]ls
[hadoop@promote .ssh]ssh localhost    输入yes

这时无需密码就可登录localhost

  • 修改 hadoop-env.sh、core-site.xml、hdfs-site.xml、mapred-site.xml、yarn-env.sh、yarn-site.xml文件

进入/etc下的hadoop

[hadoop@promote .ssh]cd ~
[hadoop@promote ~]$cd /hadoop/hadoop-2.7.7/etc/hadoop

修改hadoop-env.sh和yarn-env.sh

因为这两个文件是环境变量文件，所以修改前建议先复制

修改文件的JAVA_HOME

[hadoop@promote hadoop]$cp hadoop-env.sh hadoop-env.shark
[hadoop@promote hadoop]$vi hadoop-env.sh    --系统运行环境参数
export JAVA_HOME=/java/jdk1.8.0_201

[hadoop@promote hadoop]$cp yarn-env.sh  yarn-env.shark
[hadoop@promote hadoop]$vi yarn-env.sh  --yarn环境参数
export JAVA_HOME=/java/jdk1.8.0_201   这里需去掉注释“#”

修改.xml文件

修改此类文件，只需将变量添加到之间

修改core-site.xml

[hadoop@promote hadoop]$vi core-site.xml   --指定Hadoop的访问路径，即Hadoop的主节点服务器及端口号
<property>
<name>fs.defaultFS</name>
<value>hdfs://localhost:9000</value>   #Hadoop 的访问路径
</property>
<property>
<name>hadoop.tmp.dir</name>
<value>/hadoop/tmp</value>           #Hadoop 本地临时存放路径
</property>

修改hdfs-site.xml

[hadoop@promote hadoop]$vi hdfs-site.xml    --指定HDFS主要配置参数
<property>
<name>dfs.namenode.name.dir</name>   #指定HDFS的主节点元数据文件的本地存放路径
<value>/hadoop/name</value>
</property>
<property>
<name>dfs.datanode.data.dir</name>  #指定HDFS的数据节点数据文件的本地存放路径
<value>/hadoop/data</value>
</property>
<property>
<name>dfs.replication</name>     #指定HDFS的数据分块的备份数量
<value>1</value>
</property>

用mapred-site.xml.template生成.xml文件并进行修改

[hadoop@promote hadoop]$cp mapred-site.xml.template mapred-site.xml
[hadoop@promote hadoop]$vi mapred-site.xml  --指定MapReduce服务的配置
<property>
<name>mapreduce.framework.name</name>   #指定MapReduce所使用的外部管理框架，值为yarn
<value>yarn</value>
</property>

修改 yarn-site.xml

[hadoop@promote hadoop]$vi yarn-site.xml    --yarn服务的配置
<property>
<name>yarn.nodemanager.aux-services</name>    #指定NodeManager上运行的附属服务的类型
<value>mapreduce_shuffle</value>
</property>

  • Hadoop格式化，启动和验证

格式化

[hadoop@promote hadoop]$hadoop namenode -format

这里要找到status,如果status=0，则说明前面的配置成功，如果status非零，则前面存在错误。博主格式化时出现status=1的情况。遇到错误不要害怕，我们要学会查看信息。博主通过查看上面的错误日志，发现hdfs文件中缺少“/”，于是将其补上就可以了

启动

启动dfs和yarn

[hadoop@promote hadoop]$start-dfs.sh
[hadoop@promote hadoop]$start-yarn.sh

jps查看工作进程，如果出现以下6个进程则说明配置对了，如果少一个或多个，则前面的配置出错。如果缺少namenode、datanode，应该是hdfs文件错误，如果缺少ResourceManager应该是yarn.site.xml文件错误。修改前需将dfs和yarn停掉,运行时再启动。如果前面的6个配置文件都没有错误，然后启动dfs和yarn,查看jps依然没有变化，可以将 /hadoop目录下的data、name文件删除，重新格式化

[hadoop@promote hadoop]$jps

验证

[hadoop@promote hadoop]$ hadoop fs -mkdir /test
[hadoop@promote hadoop]$ hadoop fs -ls /
[hadoop@promote hadoop]$ cd /hadoop/hadoop-2.7.7/share/hadoop/mapreduce
[hadoop@promote mapreduce]$ls

[hadoop@promote mapreduce]$ hadoop jar hadoop-mapreduce-examples-2.7.7.jar pi 2 1000

到这里，hadoop就安装完成了。

四.Hbase的安装

hbase的安装是基于zookeeper和hadoop的安装，所以在安装前确保zookeeper和hadoop都可以正常启动，切记，zookeeper和hadoop的启动不可以在root用户下启动，否则hbase安装失败

进入root 用户创建hbase目录，并将权限赋给hadoop

[hadoop@promote conf]$su - root
[root@promote ~]#mkdir /hbase
[root@promote ~]#cd /
[root@promote /]#chown hadoop /hbase

进入hadoop用户，在hbase目录下解压缩资源包

[root@promote /]#su - hadoop
[hadoop@promote ~]$cd /hbase
[hadoop@promote hbase]$ls
[hadoop@promote hbase]$tar -zxf /software/hbase-1.2.10-bin.tar.gz

创建tmp和logs

[hadoop@promote hbase]$mkdir tmp logs

添加hbase环境变量

[hadoop@promote hbase]$vi ~/.bash_profile 
#hbase environment
HBASE_HOME=/hbase/hbase-1.2.10
PATH=$HBASE_HOME/bin:$PATH
export HBASE_HOME

source 文件

[hadoop@promote hbase]$source ~/.bash_profile 
[hadoop@promote hbase]$echo $PATH   查看hbase环境变量是否添加成功

修改hbase /conf 目录下的 hbase-env.sh和 hbase-site.xml文件

修改hbase-env.sh，将下面对应修改的地方添加进去

[hadoop@promote hbase]$cd /hbase/hbase-1.2.10/conf/
[hadoop@promote conf]$cp hbase-env.sh hbase-env.sh.bak
[hadoop@promote conf]$vi hbase-env.sh
# export JAVA_HOME=/usr/java/jdk1.6.0/
export JAVA_HOME=/java/jdk1.8.0_201/
# export HBASE_CLASSPATH=
export HBASE_CLASSPATH=/hadoop/hadoop-2.7.7/etc/hadoop
# export HBASE_LOG_DIR=${HBASE_HOME}/logs
export HBASE_LOG_DIR=/hbase/logs
# Tell HBase whether it should manage it's own instance of Zookeeper or not.
export HBASE_MANAGES_ZK=false

修改hbase-site.xml文件

[hadoop@promote conf]$vi hbase-site.xml
<property>
<name>hbase.rootdir</name>    #指定HBase数据库的存放路径
<value>hdfs://localhost:9000/user/root/hbase</value>    #注：/user/root可根据自己路径进性修改
</property>                                                 
<property>
<name>hbase.tmp.dir</name>  #指定HBase的元数据的本地存放路径
<value>/hbase/tmp</value>
</property>
<property>
<name>hbase.cluster.distributed</name>  #指定HBase的分布式集群模式为开启状态
<value>true</value>
</property>
<property> 
<name>hbase.zookeeper.quorum</name>    #指定Zookeeper的连接地址，多个用“,”分隔
<value>localhost:2181,localhost:2182,localhost:2183</value>
</property>

启动habase

[hadoop@promote conf]$start-hbase.sh

查看进程

[hadoop@promote conf]$jps

如果出现如下进程，则hbase安装正确

9488 NameNode
19313 QuorumPeerMain
21763 Jps
21477 HRegionServer
18934 QuorumPeerMain
19800 SecondaryNameNode
19625 DataNode
18987 QuorumPeerMain
20107 NodeManager
21388 HMaster
19998 ResourceManager

测试

[hadoop@promote conf]$ hbase shell
hbase(main):001:0> create 'table（表名）','id','name'

到这里伪分布式平台搭建完毕！
