Sql Server 基础篇(二)

数据完整性

代码实现

一.数据完整性

1.实体完整性

1.1实体：表的每一行数据称为一个实体

1.2实体完整性：每一行记录值唯一的、不重复的

1.3实体完整性包含标识列、主键和唯一键

标识列：系统自动生成，永远不会重复
主键：非空
唯一键：唯一，但可以为null(只能空一次)  编辑器中设置方式：右键-->索引/键-->添加-->修改名称、修改类型、确定字段
2.域完整性

2.1域：字段

2.2域完整性：为了保证字段的值是否合理和准确

2.3域完整性包含非空、类型、check约束、默认值、关系（主外键）

3.自定义完整性

3.1自定义完整性：用户用自己定义的约束规则

3.2自定义完整性包含check约束、存储过程、触发器

4.引用完整性

4.1引用完整性：一个表的某个字段的值引用自另外一个表的某一个字段，被引用的表称为主表，引用表称为从表或者外键表

4.1.1 选择外键表去创建主外键关系

4.1.2建立主外键关系的字段类型和意义必需一致

4.1.3建立关系的字段主表中必须是主键或者唯一键

4.1.4添加数据的时候先添加主表数据再添加外键表

4.1.5删除数据的时候先删除外键表数据再删除主表数据

补充

关系建立的表的级联操作：

1.不执行任何操作：该报错就报错，能删除就删除

2.级联：删除主表记录，对应的从表记录也将被删除

3.set null：删除主表，从表对应记录的字段值=null ,前提是这个字段可以设置为null

4.set default:删除主表，从表对应记录的字段值=设置的默认值，前提是这个字段已经设置默认值了

二.代码实现

1.创建约束的种类：
主键约束（primary key (PK)）

唯一键约束（unique (UQ)）

检查约束（check (CK)）

默认值约束（default (DF)）

外键约束(foregin key (FK))
2.创建约束的语法
alter table 表名
add constraint 约束名称（以简写作为前缀） 约束的说明（字段）
1.将id设置为主键
alter table Teacher
add constraint PK_Teacher_id primary key（id）

2.设置name为唯一键 
alter table Teacher
add constraint UQ_Teacher_Name unique(name)

3.设置年龄再0~100之间
alter table Teacher
add constraint CK_Teacher_Age check(age>0 and age<=100)

4为Birthday添加默认值约束
alter table Teachre                      --for 指说明为哪一个字段添加默认值
add constraint DF_Teacher_Birthday default('2020-01-01') for Birthday

5.为classid添加外键约束(级联)
alter table Teacher  --从表的某一个字段引用主表的某一个字段
add constraint FK_Teacher_classid foreign key(classid) references classes(cid)
删除级联
if exists(select * from sysobjects where name='FK_Teacher_Name')
alter table teacher drop constraint FK_Teacher_Name
with nocheck  --不检查现有输出
add constraint FK_Teacher_classid foreign key(classid) reference classes(cid)
on delete set null  (前提是该属性有空值)
