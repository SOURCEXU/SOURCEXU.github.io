Censtos系统MySQL配置

CenstOS 7 MySQL
发布日期:   2020-10-28
文章字数:   910
阅读次数:  

卸载Centos自带的MySQL方法

MySQL的坏境配置

MySQL初始化配置和启动

MySQL的系统服务配置

一.卸载Censtos自带的MySQL方法

Centos7.0 默认自带了MySQL的数据库，名称为mariadb,在安装定制版MySQL数据库时，需卸载mariadb,否则会发生内核冲突

1.关闭所有的hadoop应用、做快照（配置错误导致整个伪分布式平台毁坏时可以立即恢复到配置前）
2.进入root用户，MySQL的配置全程都在root用户下进行

3.查询mariadb状态，有的话就全部卸载
[root@promote ~]#rpm -qa |grep mariadb   查询
[root@promote ~]#yum remove 文件名        卸载

4.查询有无安装libaio软件包，一般系统中都有，没有的话可以安装
[root@promote ~]#rpm -qa |grep libaio

二.MySQL的坏境配置

1.创建mysql用户和用户组
[root@promote ~]#tail /etc/passwd   --检查是否存在mysql，这里是没有的

[root@promote ~]#groupadd mysql   --创建mysql群组
[root@promote ~]#useradd -r -g mysql mysql  --在mysql群组里创建mysql用户
2.将MySQL的软件包解压到/mysql
[root@promote ~]#mkdir /mysql   --创建mysql文件
[root@promote ~]#cd /mysql   --进入到mysql目录
[root@promote mysql]#tar -zxf /software/mysql-8.0.11-linux-glibc2.12-x86_64.tar.gz

3.创建MySQL的软链接
[root@promote mysql]#cd /usr/local
[root@promote local]#ln -s /musql/mysql-8.0.11-linux-glibc2.12-x86_64/ mysql  --软链接名称‘mysql’前要有空格

正确的软链接的创建是蓝色的，如果不是则删除链接’mysql‘重新创建

删除操作： rm mysql

4.创建MySQL的data目录并变更权限
[root@promote local]#cd /usr/local/mysql  --进入mysql目录
[root@promote mysql]#mkdir data   --创建data文件
[root@promote mysql]#chmod 770 data
[root@promote mysql]#chown -R mysql .
[root@promote mysql]#chgrp -R mysql .

5.配置坏境变量
[root@promote mysql]#vi /etc/profile

在文件末尾加入以下内容

#mysql environment
MYSQL_HOME=/usr/local/mysql
PATH=$MYSQL_HOME/bin:$PATH
export MYSQL_HOME PATH
[root@promote mysql]#source /etc/profile
查看
[root@promote mysql]#echo $PATH
[root@promote mysql]#mysql    --显示不能连接（这样的报错是对的，不用管它）

三.MySQL初始化配置和启动

1.MySQL数据库的初始化

初始化后root@localhost后会出现一串密码，这串密码需要记下来，后续需要用它来登录MySQL，如果没有密码则配置错误，需将data目录删除重新配置

[root@promote mysql]#mysqld --initialize --user=mysql --basedir=/usr/local/mysql --datadir=/usr/local/mysql/data

2.MySQL数据库的ssl_rsa安装
[root@promote mysql]#mysql_ssl_rsa_setup --basedir=/usr/local/mysql --datadir=/usr/local/mysql/data

出现下面一段信息则配置正确，如果没有则配置错误，需将data目录删除重新配置

3.MySQL数据库的安全模式启动
[root@promote mysql]#mysqld_safe --user=mysql --basedir=/usr/local/mysql --datadir=/usr/local/mysql/data &
4.MySQL数据库的启动

mysql在创建数据库前需要修改密码，否则无法正常使用。

初始登录时是需要用初始化时生成的密码登录的，我这里是’ ； ？GCyJ8et;pc‘

这里就选用登录时修改密码,这里我将密码修改为 123456

[root@promote mysql]#mysqladmin -u root -p password 123456

四.MySQL的系统服务配置

1.MySQL服务加到系统服务目录并命名为mysql
[root@promote mysql]#cp /usr/local/mysql/support-files/mysql.server /etc/init.d/mysql
2.重启MySQL服务,确认配置正确
[root@promote mysql]#service mysql restart

3.MySQL服务配置为开机自启动
[root@promote mysql]#chkconfig --add mysql
[root@promote mysql]#chkconfig --level 3000 mysql on    --这里的3000可以自定义，表示在第几个服务后开启mysql,我这里表示到第3000个服务时开启
4.重启Centos,验证MySQL服务启动
[root@promote mysql]#reboot    --重启
[hadoop@promote ~]$ps -ef |grep mysql   --查看mysql进程

MySQL配置完成~
