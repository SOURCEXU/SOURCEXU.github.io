导入文件

在根目录下创建software文件夹，将资源包导入

资源获取地址：https://pan.baidu.com/s/1EW8t5szEkG-1Uk6iqeRQbQ 提取码：2a3c

通过xftp 6 将资源包传输到 /software 目录下

一.添加环境变量

进入/etc/profile文件添加环境变量

[root@promote ]# vi /etc/profile
#java environment
JAVA_HOME=/java/jdk1.8.0_201
CLASSPATH=.:$JAVA_HOME/lib/tools.jar:$JAVA_HOME/lib/dt.jar
PATH=$JAVA_HOME/bin:$PATH
export JAVA_HOME CLASSPATH PATH

#zookeeper environment
ZOOKEEPER_HOME=/zookeeper/zookeeper-3.4.13
PATH=$ZOOKEEPER_HOME/bin:$PATH
export ZOOKEEPER_HOME PATH

#hadoop environment
HADOOP_HOME=/hadoop/hadoop-2.7.7
PATH=$HADOOP_HOME/bin:$HADOOP_HOME/sbin:$PATH
export HADOOP_HOME PATH

#hbase environment
HBASE_HOME=/hbase/hbase-1.2.10
PATH=$HBASE_HOME/bin:$PATH
export HBASE_HOME

#hive environment
HIVE_HOME=/hive/apache-hive-2.1.1-bin
PATH=$HIVE_HOME/bin:$PATH
export HIVE_HOME PATH

#Sqoop environment 
SQOOP_HOME=/sqoop/sqoop-1.99.7-bin-hadoop200
PATH=$SQOOP_HOME/bin:$PATH
export SQOOP_HOME SQOOP_SERVER_EXTRA_LIB PATH

source 文件

[root@promote ]#source /etc/profile

查看环境变量

[root@promote ]#echo $PATH

查看java 版本

java -version

二.解压文件

在根目录下创建java、zookeeper、hadoop、hbase,分别进入解压

[root@promote ]#cd /
[root@promote /]#mkdir java zookeeper hadoop hbase
[root@promote /]#cd /java
[root@promote java]#tar -zxf /software/jdk-8u201-linux-x64.tar.gz
[root@promote java]#cd /zookeeper/
[root@promote zookeeper]#tar -zxf /software/zookeeper-3.4.13.tar.gz
[root@promote zookeeper]#cd /hadoop/
[root@promote hadoop]#tar -zxf /software/hadoop-2.7.7.tar.gz
[root@promote hadoop]#cd /hbase
[root@promote hbase]#tar -zxf /software/hbase-1.2.10-bin.tar.gz

三.修改配置文件

修改 hadoop-env.sh、core-site.xml、hdfs-site.xml、mapred-site.xml、yarn-env.sh、yarn-site.xml文件

修改hadoop-env.sh和yarn-env.sh

因为这两个文件是环境变量文件，所以修改前建议先复制

修改文件的JAVA_HOME

[root@promote ~]#cd /hadoop/hadoop-2.7.7/etc/hadoop/
[root@promote hadoop]#cp hadoop-env.sh hadoop-env.shark
[root@promote hadoop]#vi hadoop-env.sh

[root@promote hadoop]#cp yarn-env.sh  yarn-env.shark
[root@promote hadoop]#vi yarn-env.sh  --yarn环境参数
export JAVA_HOME=/java/jdk1.8.0_201

修改.xml文件

修改此类文件，只需将变量添加到之间

修改core-site.xml

[root@promote hadoop]#vi core-site.xml   --指定Hadoop的访问路径，即Hadoop的主节点服务器及端口号
<property>
<name>fs.defaultFS</name>
<value>hdfs://localhost:9000</value>   #Hadoop 的访问路径
</property>
<property>
<name>hadoop.tmp.dir</name>
<value>/hadoop/tmp</value>           #Hadoop 本地临时存放路径
</property>

修改hdfs-site.xml

[root@promote hadoop]#vi hdfs-site.xml
<property>
<name>dfs.namenode.name.dir</name>   #指定HDFS的主节点元数据文件的本地存放路径
<value>/hadoop/name</value>
</property>
<property>
<name>dfs.datanode.data.dir</name>  #指定HDFS的数据节点数据文件的本地存放路径
<value>/hadoop/data</value>
</property>
<property>
<name>dfs.replication</name>     #指定HDFS的数据分块的备份数量
<value>1</value>
</property>

用mapred-site.xml.template生成.xml文件并进行修改

[root@promote hadoop]#cp mapred-site.xml.template mapred-site.xml
[root@promote hadoop]#vi mapred-site.xml
<property>
<name>mapreduce.framework.name</name>   #指定MapReduce所使用的外部管理框架，值为yarn
<value>yarn</value>
</property>

修改 yarn-site.xml

[root@promote hadoop]#vi yarn-site.xml 
<property>
<name>yarn.nodemanager.aux-services</name>    #指定NodeManager上运行的附属服务的类型
<value>mapreduce_shuffle</value>
</property>

修改hbase /conf 目录下的 hbase-env.sh和 hbase-site.xml文件

修改hbase-env.sh，将下面对应修改的地方添加进去

[root@promote hadoop]#cd /hbase/hbase-1.2.10/conf/
[root@promote conf]#cp hbase-env.sh hbase-env.sh.bak
[root@promote conf]#vi hbase-env.sh

添加如下变量

export JAVA_HOME=/java/jdk1.8.0_201/
export HBASE_CLASSPATH=/hadoop/hadoop-2.7.7/etc/hadoop
export HBASE_LOG_DIR=/hbase/logs
export HBASE_MANAGES_ZK=false

修改hbase-site.xml文件

[root@promote conf]#vi hbase-site.xml
<property>
<name>hbase.rootdir</name>    #指定HBase数据库的存放路径
<value>hdfs://localhost:9000/user/root/hbase</value>    #注：/user/root可根据自己路径进性修改
</property>                                                 
<property>
<name>hbase.tmp.dir</name>  #指定HBase的元数据的本地存放路径
<value>/hbase/tmp</value>
</property>
<property>
<name>hbase.cluster.distributed</name>  #指定HBase的分布式集群模式为开启状态
<value>true</value>
</property>
<property> 
<name>hbase.zookeeper.quorum</name>    #指定Zookeeper的连接地址，多个用“,”分隔
<value>localhost:2181,localhost:2182,localhost:2183</value>
</property>

关闭防火墙

[root@promote conf]#systemctl stop firewalld
[root@promote conf]#systemctl disable firewalld

四.克隆

在Vmware workstation中克隆两台主机，共创建三台主机

五.修改主机名和网络配置

采用远程登录的方式操作主机

修改网络配置

三台虚拟机同时进行如下操作

将 BOOTPROTO=”dhcp” 注释掉，然后添加网络，网络的主机号一般在200以上

[root@promote conf]#cd /etc/sysconfig/network-scripts/

重启网络

[root@promote conf]#systemctl restart network

修改hostname

[root@promote conf]#vi /etc/hostname或 hostnamectl set-hostname xxx
#xxx 为为主机名称

分别在三台虚拟机中进入hosts，添加ip地址和主机名

[root@promote conf]#vi /etc/hosts
192.168.1.201  cluster1-1  
192.168.1.202  cluster1-2  
192.168.1.203  cluster1-3

测试

在Vmware workstation中互ping主机名
