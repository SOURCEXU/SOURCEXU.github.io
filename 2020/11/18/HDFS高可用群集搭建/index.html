导入文件

在根目录下创建software文件夹，将资源包导入

[hadoop@promote ~]$ cd /
[hadoop@promote /]$ mkdir software 
[hadoop@promote /]$ chmod 777 software  #赋予文件夹software权限

资源获取地址：https://pan.baidu.com/s/1EW8t5szEkG-1Uk6iqeRQbQ 提取码：2a3c

通过xftp 6 将资源包传输到 /software 目录下

一.添加环境变量

进入root用户

[hadoop@promote ~]$ su root
[root@promote ]# vi /etc/profile

进入/etc/profile文件添加环境变量

[root@promote ]# vi /etc/profile
#java environment
JAVA_HOME=/java/jdk1.8.0_201
CLASSPATH=.:$JAVA_HOME/lib/tools.jar:$JAVA_HOME/lib/dt.jar
PATH=$JAVA_HOME/bin:$PATH
export JAVA_HOME CLASSPATH PATH

#zookeeper environment
ZOOKEEPER_HOME=/zookeeper/zookeeper-3.4.13
PATH=$ZOOKEEPER_HOME/bin:$PATH
export ZOOKEEPER_HOME PATH

#hadoop environment
HADOOP_HOME=/hadoop/hadoop-2.7.7
PATH=$HADOOP_HOME/bin:$HADOOP_HOME/sbin:$PATH
export HADOOP_HOME PATH

#hbase environment
HBASE_HOME=/hbase/hbase-1.2.10
PATH=$HBASE_HOME/bin:$PATH
export HBASE_HOME

#hive environment
HIVE_HOME=/hive/apache-hive-2.1.1-bin
PATH=$HIVE_HOME/bin:$PATH
export HIVE_HOME PATH

#Sqoop environment 
SQOOP_HOME=/sqoop/sqoop-1.99.7-bin-hadoop200
PATH=$SQOOP_HOME/bin:$PATH
export SQOOP_HOME SQOOP_SERVER_EXTRA_LIB PATH

source 文件

[root@promote ]#source /etc/profile

查看环境变量

[root@promote ]#echo $PATH

查看java 版本

java -version

在根目录下创建java、zookeeper、hadoop、hbase,分别进入解压

[root@promote ]#cd /
[root@promote /]#mkdir java zookeeper hadoop hbase
[root@promote /]#cd /java
[root@promote java]#tar -zxf /software/jdk-8u201-linux-x64.tar.gz
[root@promote java]#cd /zookeeper/
[root@promote zookeeper]#tar -zxf /software/zookeeper-3.4.13.tar.gz
[root@promote zookeeper]#cd /hadoop/
[root@promote hadoop]#tar -zxf /software/hadoop-2.7.7.tar.gz
[root@promote hadoop]#cd /hbase
[root@promote hbase]#tar -zxf /software/hbase-1.2.10-bin.tar.gz
[root@promote hbase]#java -version

三.关闭防火墙，更改主机名

[root@promote hbase]# systemctl disable firewalld 下次开机后自动关闭防火墙
[root@promote hbase]# systemctl stop firewalld   关闭防火墙
[root@promote hbase]# systemctl status firewalld
[root@promote hbase]# hostnamectl set-hostname xmy62-1
[root@promote hbase]#exit    退出
[hadoop@promote ~]$ su root   重新登录查看

四.配置zookeeper

[root@xmy62-1 hbase]cd /zookeeper
[root@xmy62-1 zookeeper]#ls
[root@xmy62-1 zookeeper]#mkdir data logs
[root@xmy62-1 zookeeper]#cd /zookeeper/zookeeper-3.4.12/conf
[root@xmy62-1 conf]#ll
[root@xmy62-1 conf]# cp zoo_sample.cfg zoo.cfg
[root@xmy62-1 conf]# vi zoo.cfg
修改
--注释：
#dataDir=/tmp/zookeeper
#clientPort=2181
--新增：
dataDir=/zookeeper/data
dataLogDir=/zookeeper/logs
clientPort=2181
server.1=xmy62-1:2888:3888
server.2=xmy62-2:2888:3888
server.3=xmy62-3:2888:3888

在1里面输，2和3克隆之后再输
[root@xmy62-1 conf]# echo '1' > /zookeeper/data/myid
[root@xmy62-1 conf]# cat /zookeeper/data/myid

因为这两个文件是环境变量文件，所以修改前建议先复制

修改文件的JAVA_HOME

[root@xmy62-1 conf]# cd /hadoop/

[root@xmy62-1 hadoop]# mkdir data name tmp journal
[root@xmy62-1 hadoop]# cd /hadoop/hadoop-2.7.7/etc/hadoop/

修改core-site.xml

六.Hbase配置

修改hbase /conf 目录下的 hbase-env.sh和 hbase-site.xml文件

修改hbase-env.sh，将下面对应修改的地方添加进去

[root@xmy62-1 hadoop]# cd /hbase/
[root@xmy62-1 hbase]# ls
[root@xmy62-1 hbase]# mkdir data logs
[root@xmy62-1 hbase]# cd /hbase/hbase-1.2.10/
[root@xmy62-1 hbase-1.2.10]# ll
[root@xmy62-1 hbase-1.2.10]# cd /hbase/hbase-1.2.10/conf/
[root@xmy62-1 conf]# cp hbase-env.sh hbase-env.shark
[root@xmy62-1 conf]# vi hbase-env.sh

添加如下变量

export JAVA_HOME=/java/jdk1.8.0_201/
export HBASE_CLASSPATH=/hadoop/hadoop-2.7.7/etc/hadoop
export HBASE_LOG_DIR=/hbase/logs
export HBASE_MANAGES_ZK=false

修改hbase-site.xml文件

[root@xmy62-1 conf]# vi hbase-site.xml 
<property>
<name>hbase.rootdir</name>
<value>hdfs://hadoop-ha62/user/root/hbase</value>
</property>
<property>
<name>hbase.tmp.dir</name>
<value>/hbase/tmp</value>
</property>
<property>
<name>hbase.cluster.distributed</name>
<value>true</value>
</property>
<property>
<name>hbase.zookeeper.quorum</name>
<value>xmy62-1:2181,xmy62-2:2181,xmy62-3:2181</value>  #修改成三个结点的主机号
</property>
<property>
<name>hbase.master.maxclockskew</name>
<value>60000</value>
</property>
[root@xmy62-1 conf]# vi backup-masters
xmy62-1
xmy62-2
[root@xmy62-1 conf]# vi regionservers 
xmy62-2
xmy62-3

七.关机克隆虚拟机，开机后更改主机名并重新登录

1.在Vmware workstation中克隆两台主机，共创建三台主机

2.修改主机2和3的主机名

root@xmy62-1 hadoop]# hostnamectl set-hostname xmy62-2
root@xmy62-1 hadoop]# hostnamectl set-hostname xmy62-3
root@xmy62-1 hadoop]#exit
root@xmy62-1 hadoop]#su root

3.三台虚拟机同时进行如下操作

将 BOOTPROTO=”dhcp” 注释掉，然后添加网络，网络的主机号一般在200以上

[root@xmy62-1 conf]#cd /etc/sysconfig/network-scripts/
[root@xmy62-2 conf]#cd /etc/sysconfig/network-scripts/
[root@xmy62-3 conf]#cd /etc/sysconfig/network-scripts/

4.重启网络

[root@xmy62-1 conf]#systemctl restart network
[root@xmy62-2 conf]#systemctl restart network
[root@xmy62-3 conf]#systemctl restart network

5.分别在三台虚拟机中进入hosts，添加ip地址和主机名

[root@promote conf]#vi /etc/hosts
192.168.1.201  xmy62-1
192.168.1.202  xmy62-2  
192.168.1.203  xmy62-2

测试

在Vmware workstation中互ping主机名

6.配置免密登录（第一台主机进行操作）

[root@xmy62-1 hadoop]# ssh-keygen -t rsa  #‘-t’这边有空格，一直按回车发送公钥，建议1,2,3都发
[root@xmy62-1 hadoop]# ssh-copy-id -i /root/.ssh/id_rsa.pub root@xmy62-1
[root@xmy62-1 hadoop]# ssh-copy-id -i /root/.ssh/id_rsa.pub root@xmy62-2
[root@xmy62-1 hadoop]# ssh-copy-id -i /root/.ssh/id_rsa.pub root@xmy62-3

7.查看

[root@xmy62-1 hadoop]# ssh xmy62-2
[root@xmy62-1 hadoop]# ssh xmy62-3

8.三台同步时间

[root@xmy62-1 hadoop]# hwclock -w
[root@xmy62-2 hadoop]# hwclock -w
[root@xmy62-3 hadoop]# hwclock -w

八.启动zookeeper

1.修改myid

[root@xmy62-2 zookeeper]# rm -rf /zookeeper/data/myid 
[root@xmy62-2 zookeeper]# echo '2' >  /zookeeper/data/myid
[root@xmy62-2 zookeeper]# cat /zookeeper/data/myid
[root@xmy62-3 zookeeper]# rm -rf /zookeeper/data/myid
[root@xmy62-3 zookeeper]# echo '3' > /zookeeper/data/myid
[root@xmy62-3 zookeeper]# cat /zookeeper/data/myid

2.启动zookeeper

[root@xmy62-1 hadoop]# zkServer.sh start
[root@xmy62-2 hadoop]# zkServer.sh start
[root@xmy62-3 hadoop]# zkServer.sh start
[root@xmy62-1 hadoop]#zkServer.sh status
[root@xmy62-2 hadoop]#zkServer.sh status
[root@xmy62-3 hadoop]#zkServer.sh status

出现两个follower，一个leader时即为配置成功

九.启动hadoop

1.三台启动jouanalnode

[root@xmy62-1 hadoop]# hadoop-daemon.sh start journalnode
[root@xmy62-2 hadoop]# hadoop-daemon.sh start journalnode
[root@xmy62-3 hadoop]# hadoop-daemon.sh start journalnode

2.在第一台输入格式化命令

[root@xmy62-1 hadoop]# hadoop namenode -format #'-format'这边有空格

Status应该为0

3.复制文件name

[root@xmy62-1 hadoop]# scp -r /hadoop/name/ root@xmy62-2:/hadoop/
[root@xmy62-1 hadoop]# scp -r /hadoop/name/ root@xmy62-3:/hadoop/

4.在第一台格式化zk

[root@xmy62-1 hadoop]#hdfs zkfc -formatZK

5.关闭1、2、3journalnode

[root@xmy62-1 hadoop]# hadoop-daemon.sh stop journalnode
[root@xmy62-2 hadoop]# hadoop-daemon.sh stop journalnode
[root@xmy62-3 hadoop]# hadoop-daemon.sh stop journalnode

6.在第一台上启动hadoop

[root@xmy62-1 hadoop]# start-all.sh

7.在2和3上yarn-daemon.sh start resourcemanager

[root@xmy62-2 hadoop]# yarn-daemon.sh start resourcemanager
[root@xmy62-3 hadoop]# yarn-daemon.sh start resourcemanager

8.在三个结点上分别jps查看，jps查看的内容必须与下面出现的信息一致，否则配置就是不成功的
